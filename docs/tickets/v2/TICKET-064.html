TICKET-064

Merge Timeline + Calendar into one entries surface

Open Priority: Medium

Summary

After TICKET-062 shipped the calendar (with a Week agenda), Timeline and the week-agenda overlap heavily — both are day-grouped entry lists. Timeline and Calendar are really the same thing seen two ways: "my diary" vs "my schedule." Collapse them into one entries surface with a List / Week / Month view toggle, leaving Photos as its own tab.

Where this fits

Journal tabs go from [Timeline, Photos, Calendar] → [Journal entries, Photos]. The entries tab carries a List / Week / Month toggle:

  • List = today's Timeline — the infinite logged feed (no scheduled overlay).
  • Week = the existing WeekAgenda.
  • Month = the existing MonthGrid.

Rob's calls: Calendar (Month) is the default view; the stats rail becomes a toggle available across all three views (not List-only as it is today).

What you're building

  • One entries component that owns the List/Week/Month toggle and renders the existing Timeline (as List), WeekAgenda, or MonthGrid. TICKET-062 already split MonthGrid / WeekAgenda out as standalone pieces specifically to make this slot in cleanly.
  • Collapse the FileTabs to [Journal entries, Photos]; default the entries view to Month.
  • Promote the StatsRail toggle to the shared surface so it shows in any view. The rail needs window stats for the calendar views — add summary to GET /api/v1/journal/calendar (reuse JournalStream#summary for the visible window), so "in view" counts work in Week/Month, not just List.
  • Roving-tabindex keyboard navigation for the month grid — arrow keys move focus between day cells (deferred from 062 because this ticket reworks the grid surface; day cells are individually tabbable today).
Reworks shipped code. Timeline (TICKET-059/060/061) becomes the List view. Its FilterToolbar + infinite scroll + stats rail are self-contained today; folding it under the shared surface needs care so the toolbars don't fight (List shows filters; calendar views show period nav). Keep the nav.spec.js regional-h1 contract — the page h1 stays "Journal".

Acceptance criteria

The journal shows two tabs (entries + Photos); the entries tab opens to Month by default with a List/Week/Month toggle.
List is the full infinite Timeline feed; Week/Month are the calendar views with the schedule overlay.
The stats rail toggles on in any view; "in view" counts reflect the visible window (calendar endpoint returns summary).
Arrow keys move focus between day cells in the month grid; Enter/Space opens the day popover.
Plant-scoped journal (Plant Detail) and the filters still work across views; page h1 stays "Journal".
Lint + tests + the review triad pass; Playwright covers the view switching.

TICKET-063

Journal calendar — weather pills

Open Priority: Low

Summary

Add the weather pills the calendar mockup shows — a small .wx disc in each day cell (rain / frost / sun) for the user's location. Carved out of TICKET-062, which shipped the calendar without weather because the data isn't available yet.

Why this is its own ticket — the data blocker

Our OpenMeteoClient is forecast-only. It fetches current conditions + a 7-day forward forecast — there is no historical/archive fetch (see memory reference_open_meteo_client). A month calendar is mostly past days, so past-day pills need Open-Meteo's separate archive API, which isn't wired. This is a distinct external integration that shares no code with the schedule/journal work — hence its own ticket.
Permanent horizon limit. Even fully built, pills can only ever populate past days + today + the next ~7 days. Days more than ~7 in the future have no forecast to fetch — those cells stay weatherless forever. That's a property of the data, not a scope choice; design the UI to degrade cleanly (no pill, not a fake/empty one).

What you're building

  • Extend api/app/clients/open_meteo_client.rb with a historical/archive fetch (Open-Meteo archive endpoint) for a date range at the user's lat/long.
  • Cache aggressively — historical daily weather is immutable, so it's very cache-friendly (long TTL, keyed by location + date).
  • Surface per-day weather for the calendar's visible window — either fold a weather map into GET /api/v1/journal/calendar or a small dedicated weather-by-date read. Map each day's weather code → rain / frost / sun (mockup 23's .wx variants).
  • Render the pill in the month cell's day-number row (the layout already reserves the top-right slot the today-badge uses). Mobile keeps it tiny per the mockup.

Acceptance criteria

Past days + today + the next ~7 days show a weather pill matching the day's conditions for the user's location.
Days beyond the forecast horizon show no pill (no placeholder, no fake data).
Historical reads are cached (no repeat archive calls for the same location+date).
No location set → no pills, no errors.
Lint + tests pass; client + e2e cover the pill rendering.
